#### [기존 필수 지침 유지]
- **목적:** Brute-Force(무차별 대입) 공격을 차단하고, 잠금 해제 권한 남용을 방지하는 명확한 복구 프로세스를 확립한다.
- **적용 범위:** `POST /api/v1/auth/login`, `POST /api/v1/admin/users/unlock`
- **필수 규칙:**
  - 로그인 5회 연속 실패 시 대상 사용자의 `failed_attempts`를 5로 세팅하고 `locked_until`에 현재 시각 + 30분 값을 넣는다.
  - 어드민 계정에서 수동 잠금 해제 시, 반드시 `unlock_reason`(사유) 문자열 파라미터를 필수로 받아 Audit Log에 기록한다.
- **금지 사항:** 잠금 횟수 판단을 서버 DB가 아닌 쿠키나 로컬 세션으로 하여 시크릿 모드에서 뚫리게 방치하는 행위 절대 금지.
- **예외 처리:** 어드민 계정 자체가 잠긴 경우 Super Admin이 DB 콘솔에서 직접 쿼리로 해제한다.
- **완료 기준:** 잠금 상태의 유저는 정확한 패스워드를 입력해도 "계정이 잠겼습니다. N분 뒤 재시도하세요" 문구를 HTTP 403과 함께 반환받음.
- **테스트 기준:** 실패 5회 도달 후 DB 확인, 올바른 비번 입력 시도 튕겨냄 확인, 해제 API(사유 포함) 호출 후 즉시 로그인 가능해짐 순차 확인.
- **초보자가 자주 틀리는 포인트:** 잠금 처리는 잘 해놓고, 올바른 비밀번호 입력 성공 시 `failed_attempts`를 0으로 초기화하는 로직을 빼먹어서 다음에 한 번만 틀려도 바로 잠김.
