#### [기존 필수 지침 유지]
- **목적:** L4 스위치나 모니터링 봇이 시스템 부분 장애(DB 연결 끊김 등)를 즉각 감지할 수 있는 단일 엔드포인트를 제공한다.
- **적용 범위:** `GET /api/v1/health` (Public 개방 라우트)
- **필수 규칙:**
  - 응답 JSON 형태: `{ "app": "ok", "db": "ok" | "error", "timestamp": "ISO문자열" }`
  - `db` 체크는 Prisma를 통해 `SELECT 1` 쿼리를 실행하되, 반드시 **3초 Timeout**을 걸어야 한다.
- **금지 사항:** 헬스체크 라우트를 인증 미들웨어 내부에 가두어 모니터링 툴이 401 에러를 받게 만드는 행위 금지.
- **예외 처리:** DB 핑 테스트 타임아웃 발생 시, 전체 서버가 죽지 않도록 `catch`로 잡고 `db: "timeout_error"` 상태를 HTTP 200과 함께 반환한다.
- **완료 기준:** 무인증 cURL 요청 `curl -X GET https://도메인/api/v1/health` 시 1초 내에 정상 JSON 포맷 응답.
- **테스트 기준:** 로컬 DB 컨테이너를 강제 중지하고 헬스체크 호출 시 `db: "error"`가 정상 노출되는지 확인.
- **초보자가 자주 틀리는 포인트:** `USER` 테이블 count 쿼리를 헬스체크로 썼다가 모니터링 봇이 1초마다 때려서 DB에 부하를 줌 (가장 가벼운 쿼리 사용 강제).
