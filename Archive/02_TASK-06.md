#### [기존 필수 지침 유지]
- **목적:** 문제 추적을 위한 데이터 변경 로그를 남기되, 과도한 로그 쌓임으로 인한 DB 용량 고갈과 개인정보 유출을 막는다.
- **적용 범위:** `POST`, `PUT`, `PATCH`, `DELETE` 요청을 처리하는 모든 서비스 로직, `AUDIT_LOG` 테이블
- **필수 규칙:**
  - API 비즈니스 트랜잭션 성공 직후, 비동기 호출(`void createLog()`)로 `AUDIT_LOG` 테이블에 `user_id`, `action`, `target_id`를 적재한다.
  - `password`, `token`, `email`, `phone` 명칭을 가진 페이로드 값은 DB 저장 전 무조건 정규식으로 `***` 마스킹 처리한다.
  - 매일 자정, DB 스케줄러(또는 Cron Job)를 통해 `created_at` 기준 30일이 지난 Audit Log 행은 영구 삭제(또는 S3 백업) 단계를 거친다.
- **금지 사항:** 로그 저장을 비즈니스 로직과 동일 트랜잭션으로 묶어, 로그 저장 실패 시 정상 결제/승인 건까지 롤백시켜버리는 행위 금지.
- **예외 처리:** 로그 DB 테이블에 접근 불가 상태일 경우 `console.error`로만 뱉고 유저 요청은 정상 완료시킨다 (Best Effort).
- **완료 기준:** 회원가입, 수정 등 쓰기 작업 완료 시 1초 뒤 DB 로그 테이블에 마스킹된 데이터가 확인됨.
- **테스트 기준:** 이메일과 비밀번호를 담은 Body로 API 호출 시, DB `AUDIT_LOG.payload` 컬럼에 `{"email":"***","password":"***"}`로 적재되는지 검증.
- **초보자가 자주 틀리는 포인트:** 로그 기록을 `await prisma.auditLog.create(...)`로 동기 처리하여 전체 API 응답 시간을 느리게 만듦.
