# [UI-061] Bulk Import Admin Dashboard

## 1. 문서 목적
본 문서는 **PRO ILI SMART** 프로젝트의 Bulk Import 기능을 관리하는 운영 대시보드 화면에 대한 공식 명세서이다.

본 문서는 단순히 데이터 업로드 폼을 정의하는 것이 아니라, 업로드된 **배치(Job)들의 상태를 한눈에 관제하고 부분 성공, 실패 건들을 식별하여 후속 조치(재처리, 취소 등)를 수행하기 위한 UX/UI 기준**을 제공한다. `ADM-C-001`, `ADM-Q-001`에서 수립한 처리 및 조회 원칙을 시각적으로 구체화하며, 멀티테넌시와 권한(RBAC) 제약을 UI 화면 요소 레벨에서 어떻게 통제할지 정의한다. 이 명세는 향후 프론트엔드 컴포넌트 개발과 QA 테스트의 기준 문서로 활용된다.

## 12. 감사 로그 및 보안 연계

- **UI에서의 로깅 추적**: 모든 에러 메시지 패널의 하단이나 디버그 모드에서 `trace_id`와 `job_id`를 복사할 수 있는 클립보드 아이콘을 제공한다. 이는 개발자나 L2 운영 지원팀에 신속한 이슈 전달을 돕는다.
- **고위험 액션의 UI 가드**: `[재처리]`, `[강제 취소]` 버튼 클릭 시 즉시 API를 호출하지 않는다.
  반드시 `[경고: 작업 취소 시 복구가 불가능합니다. 계속하시겠습니까?]` 형태의 **파괴적 액션 모달(Destructive Modal)** 을 띄우고 명시적인 확인을 거친다.
- **민감정보 마스킹 UX**: `Row Error`의 `raw_data` 표출 시 전화번호/주민번호 등이 서버로부터 `010-****-1234` 형태로 마스킹되어 내려오는 것을 프론트엔드는 그대로 렌더링만 하며 임의의 마스킹 해제 UI를 제공하지 않는다.
