# UI-010_audit_workspace_page.md

## 1. 문서 목적
본 문서는 PRO ILI SMART 플랫폼의 Smart Audit F1 라인에 대한 통합 작업 공간(Workspace) UI 설계 기준을 정의한다. 감사 세션(Session)의 전체 수명 주기(Draft ~ Closed/Cancelled)와 감사 리포트(Report)의 생성 및 이력 조회를 단일 흐름으로 연결하여, 사용자가 직관적으로 업무를 수행할 수 있는 실무 화면 구조를 제시한다.

## 2. UI 설계 원칙
**표 1. UI 설계 원칙**

| 원칙 | 설명 | 적용 기준 |
| :--- | :--- | :--- |
| **상태 가시성 (State Visibility)** | 현재 세션 및 리포트의 상태를 명확하게 시각화 | Draft, InProgress, Submitted 등 배지(Badge) 컬러 분리 적용 |
| **행동 제한 (Action Restriction)** | 사용자 권한(RBAC) 및 현재 상태에 따라 불가능한 액션 비활성화/숨김 처리 | 권한 없는 버튼 노출 방지 (Disabled 처리 또는 렌더링 제외) |
| **단일 작업 동선 (Single Flow)** | 목록조회 → 세션생성/편집 → 제출 → 리포트 생성 → 리포트 확인까지 자연스럽게 연결 | Workspace 내 Split View 또는 Side Panel 구조 활용 |
| **보조 수단 명시 (STT Clarity)** | STT 입력 데이터는 공식 판단이 아닌 '보조 참고 자료'임을 명확히 시각화 | STT 데이터 영역에 안내 아이콘/툴팁 배치 |

## 3. 적용 범위 정의
**표 2. 적용 범위 정의**

| 범위 | 포함 대상 | 제외 대상 (Sprint 2+) |
| :--- | :--- | :--- |
| **포함 기능 (Sprint 1)** | 세션 목록/상세/생성/편집/임시저장/제출/종료/취소, 리포트 생성(수동 트리거), 리포트 조회/이력조회 | Vision AI 자동 판독, 고급 결재 워크플로우 연동 |
| **데이터 연동** | STT Zero-UI 데이터(Read-Only), 기본 감사 로그 (Insert-only) | NC 시정조치 자동 발행 화면 |
| **접근 제어** | Tenant/Site 기반 권한 제한 (System Admin, Tenant Admin, Auditor, Viewer) | COPQ (저품질 비용) 대시보드 |

## 4. 사용자 역할별 화면 관점
**표 3. 역할별 화면 관점**

| 역할 (Role) | 노출 화면 및 허용 액션 | 제한 사항 |
| :--- | :--- | :--- |
| **System Admin** | 전체 Tenant/Site의 세션/리포트 목록 (Read-Only) | 직접 세션/리포트 생성 및 편집 불가 |
| **Tenant Admin** | 소속 Tenant 하위 Site의 세션/리포트 목록 (Read-Only) | 직접 세션/리포트 생성 및 편집 불가 |
| **Auditor** | 소속 Site의 세션 생성/편집/제출, 리포트 생성/조회 | 타 Site 데이터 접근 불가 |
| **Viewer** | 소속 Site의 Submitted/Finalized/Closed 상태 세션 및 리포트 조회 | 편집 및 생성 버튼 전체 미노출 |

## 5. 화면 구조 개요
**표 4. 화면 구조 개요**

| 영역명 (Zone) | 구성 요소 | 역할 및 목적 |
| :--- | :--- | :--- |
| **Global GNB / LNB** | 프로젝트 로고, 사용자 프로필, 1 Depth 메뉴 (Smart Audit) | 전역 네비게이션 및 Tenant/Site 컨텍스트 전환 |
| **List View (좌측/상단)** | 세션 및 리포트 통합 목록, 필터(상태, 날짜, 작성자), 검색바 | 작업 대상 목록 제공 및 상태 한눈에 파악 |
| **Detail View (우측/하단)** | 선택된 항목의 상세 정보 (세션 속성, STT 입력 내역, 리포트 결과 등) | 목록 선택 시 해당 항목의 상세 데이터 및 액션 버튼 노출 |
| **Action Bar (하단/상단)** | 임시저장, 제출, 리포트 생성, 재개, 취소 버튼 영역 | 현재 상태 및 권한에 따라 동적으로 버튼 상태(Active/Hidden) 전환 |

## 6. Session 중심 화면 흐름
**표 5. Session 중심 화면 흐름**

| 흐름 단계 | 화면 노출 및 동작 | 필요 권한 / 상태 조건 |
| :--- | :--- | :--- |
| **목록 진입** | 해당 Site의 세션 목록을 필터(기본: InProgress) 적용하여 표시 | Viewer 이상 권한 |
| **세션 생성** | [새 세션 생성] 버튼 클릭 시 모달/Slide-over로 초기 정보 입력 폼 제공 | Auditor (상태 무관) |
| **데이터 입력** | 체크리스트 작성, 사진 첨부, 텍스트 입력 폼 제공 (자동 임시저장 지원) | Auditor (Draft, InProgress) |
| **임시저장/제출** | [임시저장], [제출] 액션 동시 제공. 제출 시 검증(Validation) 수행 | Auditor (Draft, InProgress) |
| **재개 (Reopen)** | Submitted 상태에서 [재개] 버튼 노출. 클릭 시 InProgress로 변경 | Auditor (Submitted) |
| **종료/취소** | [세션 종료] / [세션 취소] 옵션 제공 (취소 시 사유 입력 모달) | Auditor (진행 중 상태) |

## 7. Report 중심 화면 흐름
**표 6. Report 중심 화면 흐름**

| 흐름 단계 | 화면 노출 및 동작 | 필요 권한 / 상태 조건 |
| :--- | :--- | :--- |
| **생성 후보 조회** | Submitted/Finalized 상태의 세션 상세 뷰에 [리포트 생성] 버튼 노출 | Auditor (Session: Submitted/Finalized) |
| **생성 대기/진행** | 버튼 클릭 시 '생성 중(Generating)' 스피너 및 상태 메시지 노출 (Polling/SSE 연동) | - |
| **완료 및 조회** | 생성 완료 후 화면 갱신 및 리포트 상세(Summary, Findings 등) 탭 활성화 | Viewer 이상 |
| **이력(버전) 관리** | Reopen 후 재제출로 리포트 갱신 시, 이전 리포트는 [과거 이력(Superseded)] 탭으로 분리 노출 | Viewer 이상 |

## 10. 버튼 / 행동 노출 기준
**표 10. 버튼 / 행동 노출 기준**

| 버튼명 | 노출 조건 (상태) | 노출 조건 (권한) | 에러/제한 처리 |
| :--- | :--- | :--- | :--- |

| **[리포트 생성]** | Submitted, Finalized | Auditor | 이미 최신 리포트 존재 시 Disabled 처리 |
