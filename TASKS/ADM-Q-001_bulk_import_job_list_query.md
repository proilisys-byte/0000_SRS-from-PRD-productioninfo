# [ADM-Q-001] Bulk Import Job List Query

## 1. 문서 목적
본 문서는 **PRO ILI SMART** 프로젝트의 Bulk Import 운영 과정에서 관리자가 대량 업로드된 Job의 상태와 결과를 조회(Query)하고 탐색하는 기준을 정의하는 공식 명세서이다.

단순한 데이터 나열이 아닌 "운영자의 빠른 상황 판단과 문제 식별"을 목적으로 설계되었다. 관리자가 여러 Job 중에서 어떤 것이 성공, 실패, 또는 부분 성공(PARTIAL_SUCCESS) 상태인지 빠르게 확인하고, 오류 발생 시 실패 원인을 심층 분석(Drill-down)할 수 있도록 목록 조회, 검색/필터, 정렬, 상세 조회 진입 기준을 명확히 한다. 이 기준은 후속 UI, 테스트, 라우트 핸들러 설계의 기반이 된다.

| **특정 기간 조회** | 월말 마감 등 특정 기간 중 업로드 검증 | 전 역할 | `created_at` BETWEEN {start} AND {end} | 기간 내 생성된 Job 반환 | |

## 9. 상태/결과 요약 조회 기준

### 표 8. 상태/결과 요약 조회 기준
| 상태/결과 | 의미 | 목록 표시 방식 | 운영 우선도 | 후속 액션 | 비고 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| `PENDING` | 대기 중 | 회색 점선 또는 로딩 스피너 | 낮음 | 대기 (새로고침) | |
| `PROCESSING` | 시스템 처리 중 | 파란색 프로그레스 바 (추정 행 기반) | 보통 | 모니터링 | 대용량의 경우 실시간 성격 |
| `SUCCESS` | 100% 정상 처리 | 녹색 체크 아이콘 | 낮음 | 완료 (추가 액션 불필요) | |

| `CANCELED` | 관리자 취소됨 | 빗금 표시 등 취소선 아이콘 | 낮음 | - | S2 반영 예정 |

## 14. 완료 기준(Definition of Done)
- **문서 완료 조건**: 관리자가 운영상 판단해야 할 핵심 정보(특히 `PARTIAL_SUCCESS` 식별)가 조회 목록과 필터 설계에 누락 없이 반영됨.
- **조회 기준 정의 완료 조건**: Tenant/Site 등 권한(RBAC) 기반의 쿼리 격리(Where 절 제약) 기준이 확립되어 보안 요건을 충족함.
- **후속 문서 전달 가능 조건**: UI 디자이너/개발자가 어떤 필터를 화면 상단에 빼고, 어떤 데이터를 그리드에 나열할지 본 문서를 기준으로 즉시 설계에 들어갈 수 있음.
- **테스트 가능 조건**: "A 테넌트 관리자는 B 테넌트의 Job 목록을 볼 수 없다", "실패 상태 필터링 시 실패건만 노출된다" 등의 명확한 QA Assert 기준이 제공됨.
