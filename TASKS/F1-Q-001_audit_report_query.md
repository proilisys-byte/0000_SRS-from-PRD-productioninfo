# [F1-Q-001] Audit Report & Session Query 처리 명세서

## 1. 문서 목적
이 문서는 PRO ILI SMART 프로젝트의 F1 Smart Audit 도메인에서 생성·관리되는 **감사 세션(Audit Session)** 및 **공식 감사 리포트(Audit Report)**에 대해, 누가, 무엇을, 어떤 기준으로 조회·검색·필터링하고 추적할 수 있는지 정의하는 **조회(Query) 기준 문서**이다.

- **문서 필요성**: 조회(Query)는 단순한 데이터 반환이 아니라, 권한과 상태를 해석하여 현장 운영자 및 관리자에게 올바른 의사결정 맥락을 제공하는 핵심 기능이다. 세션과 리포트의 목적 차이를 명확히 구분하여 데이터 노출 기준을 세워야 한다.
- **상위 기준 역할**: 본 문서는 프론트엔드 화면(`UI-010`), API 스펙(`API-003`, `F1-RH-001`), 테스트 시나리오(`TEST-F1-001/002`)를 구현하기 위한 논리적 근거(Ground Truth)로 작용한다.

## 6. 감사 세션 조회 시나리오

### 표 5. 감사 세션 조회 시나리오
| 시나리오 | 조회 조건 (필터) | 목적 | 반환 데이터 성격 |
| :--- | :--- | :--- | :--- |
| **내 진행 중 세션 찾기** | `auditor_id=me`, `state=InProgress\|PartiallySaved` | 감사자의 현장 앱 진입 시 "이어서 하기" 화면 구성 | 최소 목록 정보 + 진행도(%) |
| **제출 대기 세션 찾기** | `site_id=context`, `state=Submitted` | 리뷰어가 승인(Finalize)할 후보 목록 확인 | 전체 요약 및 미결사항 |
| **재오픈된 세션 추적** | `site_id=context`, `state=Reopened` | 반려되어 재조치가 필요한 세션 목록 파악 | 반려 사유(Reason) 필수 포함 |
| **특정 기간/템플릿 조회** | `date_range`, `template_id` | 주간/월간 감사 이행 실적 현황 파악 | 기간 내 목록 페이징 |
| **STT 보조 사용 조회** | `is_stt_assisted=true` | AI 보조 입력이 사용된 세션 필터링 (통계용) | 플래그 및 세션 메타 |

## 7. 감사 리포트 조회 시나리오

### 표 6. 감사 리포트 조회 시나리오
| 시나리오 | 조회 조건 (필터) | 목적 | 반환 데이터 성격 |
| :--- | :--- | :--- | :--- |
| **최신 공식 리포트 조회** | `site_id=context`, `is_latest=true`, `state=Generated` | 관리자가 사이트의 유효한 최종 감사 결과를 확인 | 리포트 메타, 요약, Findings |
| **특정 세션의 리포트 조회** | `session_id={id}`, `is_latest=true` | 세션 완료 후 생성된 결과물 바로 가기 링크 클릭 시 | 단건 리포트 상세 JSON |
| **무효화된 리포트 이력 조회** | `session_id={id}`, `state=Superseded` | 특정 감사의 수정 이력(Before & After)을 감사(Audit)하기 위함 | 과거 버전의 리포트 상세 (읽기 전용) |
| **부적합 포함 리포트 탐색** | `has_critical_finding=true` | 긴급 조치가 필요한 중대 결함 리포트만 필터링 | 리포트 목록 + 부적합 카운트 |
| **생성 실패/대기 리포트 추적** | `state=Generating\|Failed` | 시스템 오류로 공식 산출물이 나오지 않은 건 모니터링 | 시스템 메시지 포함 목록 |

## 11. 상태 기반 조회 해석 규칙

조회된 데이터의 상태(State)를 클라이언트(UI)가 어떻게 해석해야 하는지 정의한다.

### 표 10. 상태 기반 조회 해석 규칙
| 개체 | 상태 (State) | 조회 시 UI 해석 규칙 (운영 의미) |
| :--- | :--- | :--- |
| **Session** | `Draft`, `Ready` | 수정 및 바로 시작 가능. 타인에게는 미노출 권장. |
| **Session** | `InProgress`, `PartiallySaved` | "계속 진행하기" 활성화. 공식 결과물로 인용 금지. |
| **Session** | `Submitted` | 리뷰어에게는 "승인/반려" 액션 활성화. 감사자에게는 Read-only. |
| **Session** | `Finalized` | 락업(Lock-up). 변경 절대 불가. 리포트 생성이 완료되었거나 대기 중임을 의미. |
| **Session** | `Reopened` | 반려된 세션. 최상단 경고 배지와 함께 사유 노출, 재수정 폼 활성화. |
| **Report** | `Generating` | "리포트를 생성 중입니다. 잠시 후 새로고침 하세요." 로딩 UI 표출. |
| **Report** | `Generated` | 완전한 공식 문서. PDF 다운로드 및 상세 열람 허용. |
| **Report** | `Superseded` | 무효화된 과거 버전. 워터마크("이 문서는 무효화되었습니다") 노출 및 다운로드 경고. |
| **Report** | `Failed` | 생성 실패. 관리자 권한으로 "재시도(Retry)" 버튼 활성화. |

## 12. 최신본 / 이력본 / Superseded 조회 기준

### 표 11. 최신본 / 이력본 / Superseded 조회 기준
| 구분 | 쿼리 로직 | 설명 및 처리 방식 |
